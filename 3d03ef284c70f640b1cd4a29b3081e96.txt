Copula-Based Tail Risk Modeling — Deep Exploration

Copulas are one of the most powerful tools in quantitative finance for modeling multivariate dependence, especially in the tails of distributions. They allow us to separate marginal distributions from the dependence structure, which is critical when dealing with the fat-tailed, non-normal returns common in crypto and astro-influenced signals.

Why Copulas Matter for This Project

In the lunar_planetary_composite V5.0 framework:
• Assets like BTC and ETH show strong lower tail dependence during Full Moon + challenging planetary aspects (Mars-Saturn, Saturn-Pluto).
• Traditional correlation underestimates joint crash risk.
• Copulas let us quantify how likely simultaneous extreme losses are — directly feeding into Kelly sizing, position limits, and Atropos exit signals.

Core Concepts

1. Sklar’s Theorem: Any multivariate distribution can be decomposed into marginals + copula.
2. Tail Dependence: 
◦ Lower Tail Dependence (λ_L) : Probability that both assets crash together.
◦ Upper Tail Dependence (λ_U) : Probability that both surge together.

1. Key Copula Families: 
◦ Gaussian: Symmetric, zero tail dependence.
◦ Student’s t: Symmetric tails, good for financial data.
◦ Clayton: Strong lower tail dependence (crash clustering).
◦ Gumbel: Strong upper tail dependence (boom clustering).
◦ Frank: Weak tail dependence.

# Enhanced Copula Tail Risk Modeling
explore_copula_tail_risk <- function(signals_df, assets = c("BTC-USD", "ETH-USD", "SPY")) {
  
  # Prepare returns
  ret_wide <- signals_df %>%
    filter(Asset %in% assets) %>%
    pivot_wider(names_from = Asset, values_from = Return) %>%
    na.omit() %>%
    select(-Date)
  
  u <- pobs(as.matrix(ret_wide))   # Uniform ranks
  
  # Fit multiple copulas
  fits <- list(
    t = fitCopula(tCopula(dim = ncol(u)), u, method = "ml"),
    clayton = fitCopula(claytonCopula(dim = ncol(u)), u, method = "ml"),
    gumbel = fitCopula(gumbelCopula(dim = ncol(u)), u, method = "ml"),
    frank = fitCopula(frankCopula(dim = ncol(u)), u, method = "ml")
  )
  
  # Model selection
  aic_table <- tibble(
    Copula = names(fits),
    AIC = map_dbl(fits, AIC)
  ) %>% arrange(AIC)
  
  best_fit <- fits[[aic_table$`Copula[1]]]
  best_name <- aic_table`$Copula[1]
  
  # Tail Dependence Coefficients
  tail_metrics <- compute_detailed_tail_dependence(best_fit, best_name)
  
  # Visualization
  generate_advanced_copula_plots(best_fit@copula, best_name, u)
  
  # Risk Implications for Signals
  tail_risk_adjustment <- calculate_tail_risk_adjustment(tail_metrics)
  
  cat("=== Copula Tail Risk Summary ===\n")
  print(aic_table)
  print(tail_metrics)
  
  return(list(
    best_copula = best_name,
    tail_metrics = tail_metrics,
    adjustment = tail_risk_adjustment,
    aic_table = aic_table
  ))
}

      lower_tail > 0.40 ~ "ELEVATED",
      TRUE ~ "Normal"
    )
  )
}

Vine Copula Structures for Multi-Asset Modeling — Deep Exploration

Vine copulas (also known as pair-copula constructions or PCCs) are a powerful extension of standard copulas for modeling complex, high-dimensional dependence structures. They are particularly valuable in your financial-astrology-stats V5.0 framework because they can capture asymmetric, non-linear, and tail-dependent relationships across multiple assets (BTC, ETH, SPY, AAPL, etc.) while incorporating lunar phases, Magi linkages, Atropos events, and planetary dignities.

Why Vine Copulas Are Ideal Here

• High-dimensional dependence: Regular copulas struggle beyond 3–4 dimensions. Vines scale elegantly.
• Tail asymmetry: Financial assets often show stronger lower tail dependence (crashes) than upper tail (booms) — especially during Full Moon + challenging aspects (Mars-Saturn, Saturn-Pluto).
• Conditional modeling: Vines allow modeling dependence conditionally (e.g., how BTC-ETH dependence changes given SPY behavior during high Atropos strength).
• Integration with V5.0: Can be conditioned on lunar phase, Golden Linkage score, or dignity levels.

Vine Copula Structure

A vine is a sequence of trees:
• Tree 1: Unconditional pairwise copulas.
• Tree 2+: Conditional copulas.

Common Types:
• C-Vine (Canonical): Star-shaped — one variable is the root (good when one asset dominates, e.g., BTC).
• D-Vine (Drawable): Path-shaped — good for time series or ordered assets.
• R-Vine (Regular): Most flexible, data-driven structure.

library(VineCopula)
library(tidyverse)

explore_vine_copula <- function(signals_df, 
                                assets = c("BTC-USD", "ETH-USD", "SPY", "AAPL"),
                                vine_type = "RVine") {  # "CVine", "DVine", or "RVine"
  
  # Prepare returns matrix
  ret_wide <- signals_df %>%
    filter(Asset %in% assets) %>%
    pivot_wider(names_from = Asset, values_from = Return) %>%
    na.omit() %>%
    select(-Date) %>%
    as.matrix()
  
  u <- pobs(ret_wide)   # Uniform ranks
  
  # Fit Vine Copula
  if (vine_type == "RVine") {
    vine_fit <- RVineStructureSelect(u, familyset = NA, selectioncrit = "AIC")
  } else if (vine_type == "CVine") {
    vine_fit <- RVineCopSelect(u, familyset = NA, type = 1, selectioncrit = "AIC")
  } else {
    vine_fit <- RVineCopSelect(u, familyset = NA, type = 2, selectioncrit = "AIC")
  }
  
  # Tail Dependence Analysis
  tail_matrix <- RVineTailDep(vine_fit)
  
  # Summary
  summary_stats <- tibble(
    Vine_Type = vine_type,
    LogLik = vine_fit$`logLik,
    AIC = vine_fit`$AIC,
    Num_Parameters = vine_fit$`nparams,
    Avg_Lower_Tail_Dep = mean(tail_matrix`$lower, na.rm = TRUE),
    Avg_Upper_Tail_Dep = mean(tail_matrix$`upper, na.rm = TRUE)
  )
  
  cat("=== Vine Copula Summary ===\n")
  print(summary_stats)
  print("Pairwise Lower Tail Dependence Matrix:")
  print(tail_matrix`$lower)
  
  # Visualization
  plot_vine_structure(vine_fit, vine_type)
  
  return(list(
    vine_fit = vine_fit,
    tail_matrix = tail_matrix,
    summary = summary_stats
  ))
}

plot_vine_structure <- function(vine_fit, type) {
  # Tree structure plot
  RVineTreePlot(vine_fit, tree = 1:3)
  
  # Save contour matrices for key pairs
  png("moon_study/vine_copula_structure.png", width = 1200, height = 800)
  plot(vine_fit, edge.labels = "family")
  dev.off()
}

Gaussian Mixture Copulas — Comprehensive Exploration
Gaussian Mixture Copulas (also known as Mixture Copulas with Gaussian components) are a flexible extension of standard copulas. They model complex dependence by combining multiple Gaussian copulas with different correlation matrices and mixing weights. This is particularly powerful for your V5.0 lunar_planetary_composite system, where asset returns exhibit regime-dependent dependence (e.g., stronger lower tail dependence during Full Moon + challenging planetary aspects).
Why Gaussian Mixture Copulas Are Valuable Here
Regime Switching: Markets have different dependence structures (calm vs. crisis, New Moon vs. Full Moon, high vs. low Golden Linkage periods).
Asymmetric Tails: Mixtures can approximate heavy lower tails while maintaining Gaussian marginal flexibility.
High Dimensionality: Better scaling than single copulas and more interpretable than full vines in some cases.
Astro-Conditioning: Mixing weights can be conditioned on lunar phase, Atropos strength, or Magi Golden Linkages.
Theoretical Foundation
A Gaussian Mixture Copula is defined as:
[ C(u_1,...,u_d) = \sum_{k=1}^K w_k \Phi_\Sigma^k \left( \Phi^{-1}(u_1), ..., \Phi^{-1}(u_d) \right) ]

where:
( w_k ) = mixing weight for component ( k ) (( \sum w_k = 1 ))
( \Phi_\Sigma^k ) = multivariate Gaussian copula with correlation matrix ( \Sigma_k )
( \Phi^{-1} ) = inverse standard normal CDF

Gaussian Mixture Copulas — Comprehensive Exploration
Gaussian Mixture Copulas (also known as Mixture Copulas with Gaussian components) are a flexible extension of standard copulas. They model complex dependence by combining multiple Gaussian copulas with different correlation matrices and mixing weights. This is particularly powerful for your V5.0 lunar_planetary_composite system, where asset returns exhibit regime-dependent dependence (e.g., stronger lower tail dependence during Full Moon + challenging planetary aspects).
Why Gaussian Mixture Copulas Are Valuable Here
Regime Switching: Markets have different dependence structures (calm vs. crisis, New Moon vs. Full Moon, high vs. low Golden Linkage periods).
Asymmetric Tails: Mixtures can approximate heavy lower tails while maintaining Gaussian marginal flexibility.
High Dimensionality: Better scaling than single copulas and more interpretable than full vines in some cases.
Astro-Conditioning: Mixing weights can be conditioned on lunar phase, Atropos strength, or Magi Golden Linkages.
Theoretical Foundation
A Gaussian Mixture Copula is defined as:
[ C(u_1,...,u_d) = \sum_{k=1}^K w_k \Phi_\Sigma^k \left( \Phi^{-1}(u_1), ..., \Phi^{-1}(u_d) \right) ]

where:
( w_k ) = mixing weight for component ( k ) (( \sum w_k = 1 ))
( \Phi_\Sigma^k ) = multivariate Gaussian copula with correlation matrix ( \Sigma_k )
( \Phi^{-1} ) = inverse standard normal CDF

Vine Copula Structures for Multi-Asset Modeling — Deep Exploration

Vine copulas (also known as pair-copula constructions or PCCs) are a powerful extension of standard copulas for modeling complex, high-dimensional dependence structures. They are particularly valuable in your financial-astrology-stats V5.0 framework because they can capture asymmetric, non-linear, and tail-dependent relationships across multiple assets (BTC, ETH, SPY, AAPL, etc.) while incorporating lunar phases, Magi linkages, Atropos events, and planetary dignities.

Why Vine Copulas Are Ideal Here

• High-dimensional dependence: Regular copulas struggle beyond 3–4 dimensions. Vines scale elegantly.
• Tail asymmetry: Financial assets often show stronger lower tail dependence (crashes) than upper tail (booms) — especially during Full Moon + challenging aspects (Mars-Saturn, Saturn-Pluto).
• Conditional modeling: Vines allow modeling dependence conditionally (e.g., how BTC-ETH dependence changes given SPY behavior during high Atropos strength).
• Integration with V5.0: Can be conditioned on lunar phase, Golden Linkage score, or dignity levels.

Vine Copula Structure

A vine is a sequence of trees:
• Tree 1: Unconditional pairwise copulas.
• Tree 2+: Conditional copulas.

Common Types:
• C-Vine (Canonical): Star-shaped — one variable is the root (good when one asset dominates, e.g., BTC).
• D-Vine (Drawable): Path-shaped — good for time series or ordered assets.
• R-Vine (Regular): Most flexible, data-driven structure.

explore_vine_copula <- function(signals_df, 
                                assets = c("BTC-USD", "ETH-USD", "SPY", "AAPL"),
                                vine_type = "RVine") {  # "CVine", "DVine", or "RVine"
  
  # Prepare returns matrix
  ret_wide <- signals_df %>%
    filter(Asset %in% assets) %>%
    pivot_wider(names_from = Asset, values_from = Return) %>%
    na.omit() %>%
    select(-Date) %>%
    as.matrix()
  
  u <- pobs(ret_wide)   # Uniform ranks
  
  # Fit Vine Copula
  if (vine_type == "RVine") {
    vine_fit <- RVineStructureSelect(u, familyset = NA, selectioncrit = "AIC")
  } else if (vine_type == "CVine") {
    vine_fit <- RVineCopSelect(u, familyset = NA, type = 1, selectioncrit = "AIC")
  } else {
    vine_fit <- RVineCopSelect(u, familyset = NA, type = 2, selectioncrit = "AIC")
  }
  
  # Tail Dependence Analysis
  tail_matrix <- RVineTailDep(vine_fit)
  
  # Summary
  summary_stats <- tibble(
    Vine_Type = vine_type,
    LogLik = vine_fit$`logLik,
    AIC = vine_fit`$AIC,
    Num_Parameters = vine_fit$`nparams,
    Avg_Lower_Tail_Dep = mean(tail_matrix`$lower, na.rm = TRUE),
    Avg_Upper_Tail_Dep = mean(tail_matrix$`upper, na.rm = TRUE)
  )
  
  cat("=== Vine Copula Summary ===\n")
  print(summary_stats)
  print("Pairwise Lower Tail Dependence Matrix:")
  print(tail_matrix`$lower)
  
  # Visualization
  plot_vine_structure(vine_fit, vine_type)
  
  return(list(
    vine_fit = vine_fit,
    tail_matrix = tail_matrix,
    summary = summary_stats
  ))
}
